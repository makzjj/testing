# Canonical Pipeline Registry

This registry records what is canonical today, what is still duplicated, and what conditions must be met before older paths are removed.

| Responsibility | Canonical owner/pipeline today | Known duplicate or legacy path | Target end state | Migration trigger | Removal condition |
|---|---|---|---|---|---|
| parameter read/write/read-back verification | `gui/workspace/pages/production_parameter_controller.py` shared `ParameterDefinition` pipeline | UUID-only and PWM-only verification/write flows still exist in the same controller as compatibility wrappers; Mechanical still keeps local utility orchestration on top of shared definitions | one unified parameter pipeline for all parameter types | any parameter-related change touching UUID, PWM, PID, Ramp Down, or future parameters | all live callers use the unified parameter pipeline, regression tests pass, EEPROM behavior is validated live, and old dedicated paths have no callers |
| UUID handling | live Production workbook actions now use the shared parameter pipeline | UUID-specific wrapper APIs and workflow state still exist in `ProductionParameterController`; runtime node-info UUID inventory and Production test-profile UUID verification remain separate non-programming behaviors | UUID represented only as a parameter definition inside the unified parameter pipeline for parameter programming | next UUID behavior change | all live UUID parameter-programming callers use the unified pipeline, tests pass, live EEPROM behavior is validated, and UUID-specific wrapper paths have no remaining callers |
| PWM handling | live Production workbook actions now use the shared parameter pipeline for PWM definitions | dedicated PWM wrapper APIs and PWM-specific handling still exist in `ProductionParameterController`; Mechanical also has local utility flow | PWM represented only as a parameter definition inside the unified parameter pipeline | next PWM read/write/verify change | all live PWM parameter-programming callers use the unified pipeline, tests pass, and PWM-specific wrapper paths have no remaining callers |
